\chapter{SYSTEM DESIGN AND ARCHITECTURE} \label{c4}
\section{Design Goals}
The system design focuses on four key goals: security, usability, real-time control, and scalability. Security is achieved through multi-factor verification. Usability is achieved through guided page-level flow. Real-time control is achieved through teacher session management. Scalability is achieved through backend-driven data storage and query design.

\section{Layered Architecture}
The application is designed using three layers:
\begin{enumerate}
\item \textbf{UI Layer:} Teacher and student dashboards.
\item \textbf{Logic Layer:} Session rules, validation flow, status states.
\item \textbf{Backend Layer:} Supabase-based auth and data persistence.
\end{enumerate}

\section{Teacher Module Design}
Teacher module has a live control interface that supports:
\begin{itemize}
\item class slot setup (date and period),
\item one-tap session start,
\item OTP display and countdown timer,
\item Bluetooth broadcast status,
\item live marked students list,
\item slot-wise attendance report and student-level summary.
\end{itemize}

\section{Student Module Design}
Student module follows a guided sequence:
\begin{enumerate}
\item select class date and period,
\item enter and validate OTP,
\item complete Bluetooth check,
\item complete face verification,
\item submit attendance.
\end{enumerate}

\section{Validation Rules Design}
The system enforces strict rules:
\begin{itemize}
\item OTP must belong to an active and non-expired session.
\item Session date and period must match selected class slot.
\item Bluetooth and face verification must both pass.
\item Duplicate attendance entries for same student and slot are rejected.
\end{itemize}

\section{Conclusion}
The architecture is modular and practical for classroom use. It enables real-time operations and supports future functional expansion with minimal structural changes.
