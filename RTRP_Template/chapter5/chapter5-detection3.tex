\chapter{IMPLEMENTATION} \label{c5}
\section{Development Environment}
The Smart Attendance System is implemented in Android Studio using Kotlin. The frontend uses Jetpack Compose with Material design components. Backend operations are integrated with Supabase APIs for authentication and data storage. Asynchronous operations are handled through Kotlin Coroutines.

\section{Teacher-Side Implementation}
\subsection{Session Creation}
Teacher initiates a session by selecting class date and period and tapping Start Session. The app validates required permissions, checks Bluetooth state, generates OTP, and saves session metadata in backend with expiry.

\subsection{Live Session Control}
During active session, teacher UI displays OTP, timer, live status, and ongoing marked students list. Attendance records are refreshed periodically for real-time visibility.

\subsection{Session Termination}
Session can be ended manually by teacher or automatically on timeout. Backend session flag is updated to inactive state and live attendance monitoring stops.

\section{Student-Side Implementation}
\subsection{Session Validation}
Student enters OTP and selected class slot. The app queries backend to verify whether the code belongs to an active session matching date and period.

\subsection{Verification Flow}
After session validation:
\begin{itemize}
\item Bluetooth check validates physical proximity.
\item Face verification validates identity match.
\item Mark Attendance becomes available only when both checks pass.
\end{itemize}

\subsection{Attendance Submission}
Attendance request contains student details, class slot, session mapping, and verification status metadata. Duplicate validations are applied before insertion.

\section{Error Handling and Stability Improvements}
During implementation, key stability improvements were introduced:
\begin{itemize}
\item controlled timeout handling for network requests,
\item clear retry states for failed validation,
\item guarded button states to prevent duplicate actions,
\item silent background refresh to avoid repeated loading interruptions.
\end{itemize}

\section{Conclusion}
The implemented system fulfills the targeted attendance workflow with practical validation and robust teacher-student interactions. The module-wise implementation supports direct deployment and future extension.
